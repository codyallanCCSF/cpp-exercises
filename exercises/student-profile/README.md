# Student Profile Project

This project implements a student data management system in C++ using Object-Oriented Programming (OOP) principles.
It utilizes a multi-file structure to separate the class interface, implementation, and the execution entry point.

## System Architecture

The project is divided into three primary components:
1. Student.h: Defines the class interface, including member functions and the EnrollmentDate structure.
2. Student.cpp: Contains the implementation logic and function definitions for the Student class.
3. main.cpp: The driver file used to instantiate the class and verify object behavior.

## Commit Message Designators

This project's main file uses a standardized prefix system for commit messages to indicate the functional
state of the code at the time of the commit.


* [C]: Compiles successfully. This indicates the code is syntactically correct and the linker has
successfully resolved all dependencies.

* [U]: Unsuccessful compilation. This indicates the code is in a transitional state or contains
known errors being tracked for debugging.
